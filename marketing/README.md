# BragBit — marketing page

A standalone landing page. Plain HTML, CSS, and JS: no framework, no build step, no
server. Open [`index.html`](index.html) in a browser, or drop this folder on any static
host (GitHub Pages, Netlify, S3 — anything that serves files).

It is deliberately **not** part of the Next.js app. The app's `/` is a mode-aware
redirect for a private self-host; this page is product marketing and has no instance
behind it.

## What's here

| File                                                 | What it is                                       |
| ---------------------------------------------------- | ------------------------------------------------ |
| [`index.html`](index.html)                           | The whole page — inline CSS + JS, self-contained |
| [`screenshots/`](screenshots)                        | Real captures of the running app (2× for retina) |
| [`capture-screenshots.mjs`](capture-screenshots.mjs) | Regenerates those captures from the live app     |

The design tokens, type (Fraunces / IBM Plex), and the warm "engineering logbook"
language are lifted from [`design-mockup.html`](../design-mockup.html) and the app's
`globals.css`, so the page and the product read as one thing. It carries its own
light/dark theme, and the hero screenshot swaps with it.

## The screenshots are real

Every image is a genuine capture of the app running locally — not a mockup. They show
the seeded demo workspace ("Riley's Logbook", `demo@bragbit.local`), topped up with a
year of sample wins so the activity heatmap and week streak show a logbook in real use.
`pnpm seed:demo` on its own seeds five wins, which renders an honest but empty-looking
heatmap and a zero-week streak.

**Regenerate them whenever the UI changes** — the page's whole argument is that this is
the real thing, so a stale shot quietly becomes a false claim.

```bash
# 1. a database with the demo workspace in it
createdb bragbit_shots   # or any throwaway DB
DATABASE_URL='postgres://…/bragbit_shots' pnpm db:migrate
DATABASE_URL='postgres://…/bragbit_shots' INSTANCE_MODE=private-solo pnpm seed:demo

# 2. run the app against it in private-solo mode, then:
node marketing/capture-screenshots.mjs
```

Add extra wins across the year first if you want the heatmap and streak to look alive.

## Notes

- **No demo link.** Every call to action points at the repo, because there is no hosted
  instance. If one ever ships, the CTAs are the only thing that needs changing.
- **The feature inventory is static HTML**, not generated by JS: it's the most
  keyword-dense content on the page, so it belongs in the source where crawlers and
  reader modes can see it. The headline count is counted off the DOM at runtime, so it
  can't drift from the list.
- **Nothing is hidden without JS.** The scroll reveals are scoped to `html.js`, which an
  inline script sets — a reveal animation that fails closed would leave the page blank.
- **Keep the inventory honest.** Everything listed ships on this branch. A landing page
  that overstates the product is a bug.
